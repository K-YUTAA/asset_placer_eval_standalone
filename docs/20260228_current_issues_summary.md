# 現状の問題点まとめ（2026-02-28）

## 目的
現時点のパイプライン（`openai -> spatial understanding -> rule-based -> eval/refine`）で、再現性・精度・評価整合に関する主要課題を整理する。

## 優先度A（最優先で対処）

### 1. 壁貫通・壁際不整合が一部ケースで残る
- 症状:
  - 可視化上、家具や経路が壁を跨いで見えるケースがある。
  - `plot_with_bg` と評価器の占有判定が一致しない見え方が出る。
- 影響:
  - 評価指標（`C_vis_start`, `R_reach`, `clr_min`）の信頼性が落ちる。
  - refine の改善結果を正しく比較できない。
- 原因候補:
  - 連続座標とグリッド占有の丸め差。
  - inner frame/outer boundary の基準差。
  - ドア・窓投影後の境界更新順序。
- 対応方針:
  - 「評価器の占有グリッド」を唯一の真値として、描画側を同じ判定に寄せる。
  - 壁境界の安全マージン（epsilon）を全ステップで統一。

### 2. Proposed refine の改善量が小さい
- 症状:
  - heuristic と比べて改善はあるが、体感で「大きく良くなった」と言いづらい。
- 影響:
  - 手法差分の主張が弱い。
- 原因候補:
  - 目的関数の重み/優先順位が現ケースに対して弱い。
  - 局所探索範囲（候補家具・深さ）が不足。
  - ハード制約が強すぎて有効解に到達しにくい。
- 対応方針:
  - 辞書式優先順位の見直し（entry改善を上位化）。
  - 候補家具選択ロジックの再設計（入口遮蔽寄与・経路寄与を明示加点）。

## 優先度B（精度安定化）

### 3. ドア検出の位置・幅がケース依存で不安定
- 症状:
  - sliding door の開口部位置がずれるケースが残る。
- 影響:
  - start点生成、経路評価、最終配置の整合が崩れる。
- 原因候補:
  - Step1アンカーとのマッチング競合。
  - 同種ドア複数時の割当順依存。
- 対応方針:
  - Gemini検出を座標真値に寄せ、割当は「最寄り壁＋最短距離＋タイプ整合」で決定。
  - 入口ドアは別IDで固定ルール化（主開口を優先）。

### 4. room_inner_frame の安定性（特にメイン領域）
- 症状:
  - 一部ケースで main inner frame が過小/過大になる。
- 影響:
  - 家具の許容配置領域が誤る。
- 原因候補:
  - サブルーム分離と最大矩形抽出の競合。
  - 境界線追従の閾値がケース依存。
- 対応方針:
  - dimensions由来の部屋数・外形比を制約として prompt/後処理に反映。
  - coverage 下限を割った場合のみ inner_frame 再取得フォールバック。

## 優先度C（研究評価の見せ方）

### 5. Stressケースの妥当性説明をさらに強化する必要
- 症状:
  - 「劣化はしているが意図どおり壊れているか」が見た目だけでは伝わりにくい。
- 影響:
  - 評価改善の説得力が弱まる。
- 対応方針:
  - 各 stress ケースに「どの指標を落とす狙いか」を manifest に明記。
  - before/after の指標差分をケース毎に併記。

### 6. OOE（entry観測）指標の主指標定義が未凍結
- 症状:
  - `hit` と `surf` のどちらを主指標にするか揺れやすい。
- 影響:
  - Adopt判定と改善比較の解釈がぶれる。
- 対応方針:
  - `eval_v1` で primary 指標を固定し、もう片方は補助ログに統一。

## 直近の実行方針（短期）
1. 壁境界と占有判定の単一化（描画と評価器の同一ルール化）。
2. ドア割当ルールの再固定（Gemini座標優先、入口ドア優先ID）。
3. inner_frame の再取得フォールバック条件を明文化・実装。
4. その後に refine（heuristic/proposed）を再比較し、差分を再評価。
